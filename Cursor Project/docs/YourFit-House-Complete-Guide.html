1. What You Have

YourFit House is a clothing brand website with:

  • Public store — homepage with collections and products
  • Admin panel — add, edit, delete items
  • SQLite database — saves products, collections, newsletter emails, and login sessions
  • GitHub-ready — repo name yourfit

2. Requirements

RequirementDetails

You do not need to run npm install if you use ./start.sh (uses Node’s built-in SQLite).

4. Step 1 — Start the Server (Local)

  1. Open Terminal.
  2. Go to your project folder: 
    cd "/Users/shivyadav/Documents/Cursor Project"
  3. Start the server: 
    ./start.sh
  4. Leave this Terminal window open. Closing it stops the site.
  5. You should see: 
    YourFit House running at http://localhost:3000
Admin panel: http://localhost:3000/admin/
SQLite database: .../data/yourfit.db
Login: admin / yourfithouse
Do not use python3 -m http.server 8080 for admin or database — that only shows static files. Always use ./start.sh on port 3000.

5. Step 2 — Open the Storefront

  1. With the server running, open your browser.
  2. Go to: http://localhost:3000
  3. You should see the YourFit House homepage with collections and products loaded from the database.

6. Step 3 — Admin Login

7. Step 4 — Add Shop Items

Shop items appear on the homepage under “Fresh from the house”.

  1. In admin, click Shop items in the sidebar (or + Add shop item on Dashboard).
  2. Click + Add shop item.
  3. Fill in the form:
    • Item name — e.g. House Logo Tee
    • Details — e.g. Heavyweight cotton · 4 colors
    • Price ($) — e.g. 42
    • Badge — New, Hot, or None
    • Image — upload a file OR paste an image URL (https://...)
  4. Click Save.
  5. Open http://localhost:3000 and refresh — your item appears on the store.

8. Step 5 — Add Collections

Collections appear on the homepage under “Collections”.

  1. Click Collections in the admin sidebar.
  2. Click + Add collection.
  3. Fill in: Title, Description, Card size (Normal or Large featured), Sort order, Image.
  4. Click Save and refresh the store.

9. Step 6 — Database (SQLite)

All data is stored in one file:

data/yourfit.db
TableStores
productsShop items
collectionsCollection cards
adminLogin username & encrypted password
sessionsLogin sessions (stay logged in after restart)
newsletter_subscribersEmail signups from homepage

Sessions last 7 days. Data persists when you restart the server.

11. Step 8 — GitHub Pages (Preview)

  1. After pushing code, open your yourfit repo on GitHub.
  2. Go to Settings → Pages.
  3. Under Build and deployment, set Source to GitHub Actions.
  4. Wait for the “Deploy GitHub Pages” workflow to finish (Actions tab).
  5. Your preview URL: https://shivtumca12.github.io/YourFit/
GitHub Pages limitation: Only the storefront design is shown. Admin login, adding items, and the database require the Node server (run ./start.sh locally, or deploy to Render/Railway for a full online app).

13. Troubleshooting

Connection refused / site won’t load

  • Server is not running — run ./start.sh and keep Terminal open.
  • Use http://localhost:3000 (not 8080).
  • Use http:// not https://.

Login screen doesn’t go away after sign in

  • Hard refresh: Cmd + Shift + R
  • Confirm you’re on port 3000 with ./start.sh running.

Products don’t show on homepage

  • Must use Node server (./start.sh), not Python static server.
  • Add items in admin under Shop items, then refresh the store.

Could not load products / collections

  • Start ./start.sh first, then reload the page.

Stop the server

In the Terminal where ./start.sh is running, press Ctrl + C.
